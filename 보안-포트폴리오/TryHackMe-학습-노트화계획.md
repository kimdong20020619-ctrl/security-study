# 📝 노트화 계획

## 원칙 (검증됨)

- **기존 노트 형식 유지** ([[tryhackme/README]] 참고): 개념 → 표 → 실습 기록(실제 답·플래그) → **배운 점 3가지** → 용어 → 관련 노트(위키링크).
- **허구 금지**: 실습 답·플래그·명령 결과는 **본인이 실제로 얻은 값만**. 안 푼 건 비워둔다.
- **커밋은 사용자가 직접** (자동 커밋 금지).

## 폴더 구조 (신설 예정)

```
tryhackme/
├── README.md                 # 진행표 (갱신)
├── pre-security/             # ✅ 완료
├── cyber-security-101/       # SEC1, 진행 중 → 모듈별 노트
├── soc-level-1/              # SOC1, 완주 후 일괄 노트화 (모듈 1~10)
└── soc-level-2/              # SOC2, 2027
```

## 타이밍 규칙

| 대상 | 노트화 시점 |
|------|------------|
| **SEC1 (진행 중)** | 모듈 1개 완료할 때마다 즉시 |
| **SOC1** | **완주 후 한 번에** (사용자 요청대로), 모듈별 노트 10개 |
