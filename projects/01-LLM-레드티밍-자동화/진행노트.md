## 한 줄 설명
LLM 서비스에 자동으로 다양한 탈옥(jailbreak)·프롬프트 인젝션 페이로드를 주입해 취약점을 찾아내고 리포트로 정리하는 공격 시뮬레이션 도구.

## 브레인스토밍 결정 사항

| 항목 | 결정 |
|------|------|
| 공격 대상 | 직접 만든 더미 챗봇 (완전히 통제된 환경, 윤리/ToS 문제 없음) |
| 마일스톤 구조 | 1차: 게임형 챗봇(비밀번호 사수) → 2차: 회사 내부 정책봇(기밀 키워드 사수). 프레임워크는 공유, 타겟 설정만 교체 |
| 타겟 챗봇 LLM | 로컬 무료 모델 (Ollama, Llama 3 / Mistral 등) — 비용 없이 대량 반복 테스트 |
| 공격 페이로드 | 공개된 알려진 탈옥 기법(DAN, 역할극, 인코딩 우회 등)으로 시작 → 이후 직접 변형/창작 페이로드 추가 |
| 성공 판정 | 1차: 키워드/패턴 매칭 (구현 단순, 명확) → 2차: LLM-as-judge 추가 (문맥상 유출 판단) |
| 리포트 형식 | 개발 중: 터미널 텍스트 요약 → 최종 목표: HTML/Markdown 공유용 리포트 |
| 공격 방식 범위 | Single-turn만 (한 번의 메시지로 판정). Multi-turn 대화형 공격은 향후 확장 과제로 보류 |
| 아키텍처 | 모듈형 인터페이스 설계 — Target / Judge / Reporter를 독립 컴포넌트로 분리, 마일스톤 전환 시 설정만 교체 (기간이 걸리더라도 정석대로 진행하기로 결정) |

상세 아키텍처·데이터 흐름·config 예시·에러 처리·테스트 전략은 [[설계-스펙]] 참고.
