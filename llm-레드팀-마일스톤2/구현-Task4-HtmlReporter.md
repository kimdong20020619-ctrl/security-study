# 구현: Task4 — HtmlReporter (+ 보안 수정)

허브: [[허브]] · 설계: [[설계-HtmlReporter]]

## 어떻게 진행했나
서브에이전트 위임(Sonnet) + TDD. 1차 구현 후 리뷰에서 **HTML 인젝션 취약점** 발견 → 별도 fix 서브에이전트로 즉시 수정 → 재리뷰.

## 발견된 문제
`_build_html()`가 페이로드 텍스트·LLM 판정 상세(`detail`)를 이스케이프 없이 HTML에 그대로 삽입. 이 도구 특성상 페이로드 자체가 "공격자가 작성한 텍스트"이고 LLM 응답도 임의 텍스트를 반향할 수 있어, `<script>` 같은 태그가 섞이면 브라우저에서 report.html을 열 때 깨지거나 마크업으로 실행될 위험.

## 수정
`html.escape()`를 카테고리·페이로드 텍스트·판정 상세 4곳에 적용. 정적 라벨(성공/차단 등)·CSS는 이스케이프 제외. `<script>alert(1)</script>`를 실제로 렌더링해서 원문이 안 남고 이스케이프된 형태만 남는지 검증하는 테스트 추가.
