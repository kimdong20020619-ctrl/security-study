# 모듈 1 — 사이버 보안 소개

---

## 섹션 1 — 공격적 보안 (Offensive Security)

공격적 보안 = 실제 해커보다 먼저 취약점을 찾기 위해 공격자의 입장에서 생각하는 것

- 해커의 행동을 모방하여 시스템 취약점을 찾는 것 = **공격적 보안 (Offensive Security)**

---

### dirb

웹사이트 주소에 단어 목록을 하나씩 붙여서 숨겨진 페이지를 찾는 도구로,  
자동으로 수천 개의 단어를 대입해 실제로 존재하는 페이지를 찾아내는 명령어

---

## 섹션 2 — 방어적 보안 (Defensive Security)

방어적 보안 = 장치와 시스템을 방어 및 보호 / 공격을 탐지하고 조사하며,  
피해가 발생하기 전에 대응하는 데 중점을 두고 모니터링하고 보호하는 것

- **주요 목표** = 공격을 탐지하고 대응 (Detect and respond to attacks)

### 모니터링 도구

- 장치에서 어떤 활동이 일어나고 있는지에 대한 정보 제공
- 대시보드 도구를 사용해 시스템 활동 확인 후 의심스러운 활동 조사 가능
- 공격자가 웹사이트에서 무엇을 찾으려 했는지 기록 확인 가능
- 공격자가 무엇을 달성하려 했는지 파악 후 조치 및 근본 문제 해결 가능

### 대응 방법

1. **IP 주소 차단** — 공격자의 기기가 시스템에 접근하는 것을 차단
2. **접속 제한 적용** — 특정 시간 동안 모든 사용자의 접속 횟수 제한
3. **보안 규칙 업데이트** — 공격자가 침입했던 중요 페이지에 대한 접근 제어 강화

## 섹션 3 — 사이버 분야 직업

| 직업 | 설명 |
|------|------|
| **보안 분석가** | 잠재적인 보안 사고를 조사·종합하여 조치 판단 및 대응, 조직 방어에서 중요한 역할 담당 |
| **보안 엔지니어** | 네트워크와 장치를 보호하는 시스템·프로세스 구축 및 유지 관리, 사이버 보안의 설계자 역할 (IDS 등) |
| **사고 대응자 (Incident Responder)** | 보안 사고 발생 시 신속하게 대응하고 피해를 최소화하는 역할. 사고 대응 절차와 정책을 수립하고 실행한다. |
| **디지털 포렌식 전문가** | 사이버 범죄의 디지털 증거를 수집·분석하는 역할. 파일 시스템, 네트워크 로그, 메모리 등을 분석한다. |
| **악성코드 분석가** | 악성 소프트웨어를 분석해 동작 원리와 위협을 파악하는 역할. 정적 분석과 동적 분석 기법을 사용한다. |
| **침투 테스터** | 펜테스팅/윤리적 해킹, 기업 시스템에 침입해 보안 수준 점검, 위임 계약 기반으로 진행 |
| **레드 팀원 (Red Teamer)** | 침투 테스터와 유사하지만 더 현실적인 공격 시나리오로 장기간 침투를 시도하며 조직의 탐지·대응 능력까지 평가한다. |

> 목표 직군: **보안 솔루션 기술지원 SE** (신입) → 경력 후 **기업 보안 담당** 또는 **악성코드 분석가**

## 실습 기록

### 실습 — FakeBank 웹사이트 숨겨진 페이지 탐색

**사용 도구**: dirb  
**목표**: 눈에 보이지 않는 숨겨진 경로를 탐색하여 민감한 기능 페이지 발견

**실습 과정**

1. dirb로 FakeBank 웹사이트의 숨겨진 주소 탐색
2. 숨겨진 송금 페이지 발견 후 접속
3. 해당 페이지에서 처음 접속 시 보였던 계좌 주소로 $2000 이체

**배운 점**

1. 눈에 보이는 페이지 이외에 숨겨진 페이지가 존재할 수 있음
2. dirb는 단어 목록을 자동으로 대입해 숨겨진 경로를 찾아냄
3. 공격자는 이런 방식으로 관리자 페이지나 민감한 기능을 찾아낼 수 있음
