# 모듈 3 — 운영 체제 기초 (Operating Systems Basics)

---

## 섹션 1 — 운영 체제: 소개

### 운영 체제란

운영 체제(OS)는 컴퓨터에서 일어나는 모든 일을 조율하는 소프트웨어입니다.
사용자, 애플리케이션, 시스템의 물리적 하드웨어 사이에 위치하여 전체 컴퓨터가 하나의 통합 시스템으로 작동하도록 유지하는 관리자 역할을 합니다.

---

### 시스템 권한 계층

- **커널 공간 (Kernel Space)**
  - 시스템의 특권적이고 안전하게 보호되는 핵심 영역
  - 운영 체제의 핵심 구성 요소인 커널이 실행되는 곳으로, 하드웨어와 시스템 리소스를 직접 관리한다.
  - 커널은 하드웨어 및 시스템 리소스에 대한 무제한 접근 권한을 가지고 있다.

- **사용자 공간 (User Space)**
  - 모든 표준 애플리케이션이 실행되는 영역
  - 사용자 공간의 애플리케이션은 하드웨어에 직접 접근할 수 없도록 의도적으로 제한된다.
  - 파일을 열거나 저장하고, 소리를 재생하거나, Wi-Fi에 연결해야 할 때마다 **시스템 호출(System Call)** 을 통해 커널에 작업을 요청해야 한다.

---

### 운영 체제의 역할

| OS 책임 | OS 역할 | 예시 |
|---------|---------|------|
| **프로세스 관리** | 프로그램을 생성, 예약, 우선순위 지정 및 종료한다. OS는 각 프로세스에 CPU 시간을 얼마나 할당할지 결정하여 멀티태스킹을 원활하게 한다. | 컴퓨터가 멈추지 않고 브라우저, 음악 플레이어, 소셜 미디어 등 여러 앱을 동시에 실행할 수 있다. |
| **메모리 관리** | RAM을 할당하며, 다른 프로세스로부터 앱의 메모리를 보호하고 앱이 종료될 때 메모리를 회수한다. RAM이 부족해지면 OS는 시스템 안정성을 유지하기 위해 가상 메모리를 사용한다. | 여러 앱을 동시에 열면, OS는 각각에 RAM을 할당하고 분리하여 서로 간섭하거나 충돌하지 않도록 한다. |
| **파일 시스템 관리** | 파일을 디렉터리로 정리하고, 이름 지정, 경로, 권한, 메타데이터(이름, 크기, 유형, 타임스탬프)를 처리한다. | 새 폴더 만들기, 사진 저장하기, 파일을 "읽기 전용"으로 설정하기 |
| **사용자 관리** | 여러 사용자 계정, 인증 및 권한을 관리하여 누가 무엇에 접근할 수 있는지 결정한다. | 비밀번호로 로그인하여 다른 사용자 계정의 파일에 접근할 수 없도록 보호한다. |
| **장치 관리** | 드라이버를 로드하고 범용 인터페이스(하드웨어 추상화 계층)를 제공하여 앱이 "이것을 출력해" 또는 "이 소리를 재생해"라고 요청할 수 있도록 한다. | 새 마우스, 프린터 또는 외장 하드 드라이브를 연결하면 즉시 작동한다. |

---

### 운영 체제의 보안

OS는 보안의 기반 역할을 합니다. 안티바이러스 프로그램이나 방화벽 같은 도구가 추가되기 전에도, OS는 이미 백그라운드에서 다음과 같은 보호 조치를 시행하고 있습니다.

| 보안 기능 | 설명 |
|----------|------|
| **인증 (Authentication)** | 로그인 비밀번호와 생체 정보를 통해 사용자의 신원을 확인한다. |
| **권한 (Authorization)** | 각 사용자와 앱이 읽고, 쓰고, 실행할 수 있는 내용을 정확하게 제어한다. |
| **격리 (Isolation)** | 모든 프로세스를 자체적으로 보호된 환경(커널/사용자 공간 분리)에 유지한다. |
| **시스템 보호 (System Protection)** | 중요한 시스템 파일과 설정을 무단 변경으로부터 보호한다. |

---

### OS 인터페이스

- **그래픽 사용자 인터페이스 (GUI, Graphical User Interface)**
  - 컴퓨터에서 접근하고 싶은 모든 정보를 그래픽으로 표현한다.
  - 폴더 아이콘, 애플리케이션 창, 설정 메뉴 등이 예시이다.
  - 내비게이션 앱과 비슷하게 생각할 수 있다. 방문하고 싶은 장소의 아이콘을 탭하면 앱이 길 안내를 생성해 주므로 직접 명령을 입력할 필요가 없다.

- **명령줄 인터페이스 (CLI, Command Line Interface)**
  - 아이콘을 클릭하는 대신, 시스템이 이해하는 단어와 구문을 사용하여 컴퓨터에 원하는 바를 정확하게 알려준다.
  - 고급 작업에서 훨씬 더 높은 정밀도, 제어력, 속도를 얻을 수 있지만 명령어에 익숙해져야 한다.
  - 지도 비유로 보면, CLI는 목적지의 정확한 GPS 좌표를 직접 입력하는 것과 같다. 직접적이고 매우 정확하지만 입력 정보가 정확해야 한다.

### 운영 체제 환경

| OS 유형 | 주요 사용 사례 | 주요 특징 |
|---------|--------------|---------|
| **데스크톱 (Desktop)** | 개인용 컴퓨터, 일상 업무, 게임, 콘텐츠 제작 | 풍부한 그래픽 인터페이스, 여러 앱을 동시에 실행 가능, 사용자 중심적 |
| **서버 (Server)** | 웹 호스팅, 데이터베이스, 클라우드 서비스, 백엔드 | GUI가 없는 헤드리스 환경, 최대 가동 시간, 다중 사용자, 원격 접속 |
| **모바일 (Mobile)** | 스마트폰과 태블릿 | 터치 기반 UI, 전력 효율성, 상시 연결, 앱 샌드박싱 |
| **임베디드 (Embedded)** | 가전제품, 자동차, IoT 기기, 스마트 TV, 라우터 | 제한된 하드웨어에서 소용량으로 특정 기능만 수행 |
| **가상/클라우드 (Virtual/Cloud)** | 실험실 장비, 컨테이너, 클라우드 인스턴스 | 경량화, 확장성, 빠른 배포 |

### 주요 용어 정리

| 용어 | 설명 |
|------|------|
| **운영 체제 (OS)** | 하드웨어, 애플리케이션 및 모든 시스템 리소스를 관리하는 핵심 소프트웨어 |
| **커널 공간 (Kernel Space)** | 하드웨어에 직접 접근할 수 있는 높은 권한을 가진 OS의 핵심 영역. 커널이 위치하며 하드웨어 및 시스템 리소스를 직접 관리한다. |
| **사용자 공간 (User Space)** | 보안 및 시스템 안정성을 위해 제한된 권한을 가진 영역. 일반 애플리케이션이 실행되는 곳이다. |
| **GUI (그래픽 사용자 인터페이스)** | 창, 아이콘, 메뉴 등 시각적 요소를 통해 클릭과 탭으로 시스템과 상호 작용하는 인터페이스 |
| **CLI (명령줄 인터페이스)** | 명령어를 직접 입력하여 시스템을 정확하고 빠르게 제어하는 텍스트 기반 인터페이스 |

## 섹션 2 — 윈도우 기초 (Windows Basics)

### 1. 로그인 및 인증

| 계정 유형 | 설명 |
|----------|------|
| **게스트 (Guest)** | 임시 접속용으로 제한된 계정. 최소한의 권한만 부여되며 시스템 설정을 변경할 수 없다. |
| **표준 (Standard)** | 시스템 전체 변경 권한 없이 애플리케이션 실행 및 개인 설정 변경과 같은 일상적인 작업을 위한 사용자 계정 |
| **관리자 (Administrator)** | 소프트웨어 설치, 구성 변경 및 사용자 관리를 포함하여 시스템에 대한 모든 권한을 가진 계정 |

---

### 2. 윈도우 데스크톱 (Windows Server 2019 기준)

- **바탕화면**: 파일, 폴더 및 바로가기가 저장되는 주요 작업 공간
- **작업 표시줄**: 애플리케이션, 시스템 도구, 설정 및 알림에 접근할 수 있는 제어 막대

**최신 윈도우 버전 구성 요소**

| 구성 요소 | 설명 |
|----------|------|
| **바탕화면 아이콘** | 휴지통, 폴더, 자주 사용하는 애플리케이션 등의 바로가기 아이콘. 모든 기능을 자유롭게 사용자 지정 가능 |
| **시작 메뉴** | 애플리케이션, 설정 및 전원 옵션에 접근하는 주요 경로. 로그아웃, 재시작 또는 컴퓨터 전원 끄기 가능 |
| **검색** | 키워드를 사용하여 애플리케이션, 파일, 폴더 및 시스템 설정을 빠르게 찾을 수 있다. |
| **작업 보기** | 현재 열려 있는 모든 창을 확인하고 빠르게 전환 가능 |
| **고정된 애플리케이션 및 폴더** | 가장 자주 사용하는 애플리케이션과 폴더를 고정할 수 있다. |
| **네트워크 및 오디오 설정** | 필요에 따라 사용자 지정할 수 있다. |
| **날짜 및 시간** | 전체 달력이 열리며, 날짜 및 시간 설정 가능 |
| **알림** | 컴퓨터 또는 애플리케이션 알림이 표시. 네트워크 및 기타 설정에도 접근할 수 있다. |

---

### 3. 윈도우 구성 및 보안

#### 업데이트

**윈도우 업데이트**
- 내장 업데이트 도구가 포함되어 있어 OS를 최신 상태로 유지한다.
- 일부 기본 애플리케이션과 보안 기능도 최신 상태로 유지된다.
- 설정 앱을 통해 접근할 수 있으며, 구성에 따라 자동으로 업데이트가 설치될 수 있다.
- 일반적으로 보안 패치, 성능 개선 및 버그 수정이 포함된다.

**애플리케이션 업데이트**
- 내장 애플리케이션은 백그라운드에서 자동으로 업데이트될 수 있다.
- 타사 애플리케이션에는 자체 업데이트 메커니즘이 포함되어 있는 경우가 많다.
- 일부 애플리케이션은 실행 시 업데이트하라는 메시지를 표시한다.
- 일부 앱은 업데이트를 확인하거나 새 설치 프로그램을 수동으로 다운로드해야 한다.

#### 애플리케이션 설치

- **Microsoft Store**: 엄선되고 안전한 앱을 Windows에 설치할 수 있는 옵션을 제공하지만, Windows Server에서는 기본적으로 사용할 수 없다.
- **직접 다운로드**: 신뢰할 수 있는 공급업체의 웹사이트에서 설치 프로그램을 직접 다운로드하여 설치한다. 설치 파일은 일반적으로 `.exe` 또는 `.msi` 형식으로 제공되며, 설치 과정을 안내한다.

#### 애플리케이션 제거

- Microsoft Store를 통해 설치된 애플리케이션 제거
- 시스템 설정의 프로그램 추가/제거 기능
- 제어판의 프로그램 제거 섹션
- 애플리케이션에 내장된 제거 프로그램 사용

#### 설정

- **Windows 설정**: 시스템, 장치, 개인 설정 및 보안 설정을 구성할 수 있는 최신 중앙 집중식 인터페이스
- **제어판**: 특정 관리 작업을 위해 여전히 필요한 기존 관리 인터페이스. Windows 설정과 함께 시스템 설정을 확인하고 수정하는 데 사용된다.

#### 작업 관리자

| 탭 | 설명 |
|----|------|
| **프로세스** | 현재 실행 중인 앱 및 백그라운드 프로세스와 해당 리소스 사용량 |
| **성능** | CPU, 메모리, 네트워크 등 시스템 리소스에 대한 그래프 및 통계 |
| **사용자** | 현재 로그인한 사용자 및 사용 중인 리소스 |
| **상세 정보** | 실행 중인 프로세스에 대한 기술적인 보기로, 프로세스 ID(PID)가 포함된다. |
| **서비스** | Windows 서비스 및 현재 상태(실행 중 또는 중지됨) |

#### Windows 기본 보안

| 기능 | 설명 |
|------|------|
| **바이러스 및 위협 방지** | 실시간 보호 및 사용자 지정 가능한 검사를 통해 악성 소프트웨어를 탐지하고 제거하는 데 도움을 준다. |
| **방화벽 및 네트워크 보호** | 무단 접근을 방지하기 위해 들어오고 나가는 네트워크 트래픽을 제어한다. |
| **앱 및 브라우저 제어** | 잠재적으로 위험한 앱, 파일 및 웹사이트로부터 사용자를 보호한다. |
| **장치 보안** | 시스템 보안을 강화하는 하드웨어 기반 보호 기능을 제공한다. |

#### 윈도우 디펜더 방화벽 (Windows Defender Firewall)

무단 네트워크 트래픽으로부터 시스템을 보호하는 방화벽으로, 네트워크 유형에 따라 세 가지 프로필로 구분된다.

| 프로필 | 설명 |
|--------|------|
| **도메인 (Domain)** | 시스템이 조직의 도메인 네트워크에 연결될 때 사용된다. |
| **개인용 (Private)** | 가정이나 실험실 환경과 같이 신뢰할 수 있는 네트워크에서 사용하도록 설계되었다. |
| **공용 (Public)** | 공용 Wi-Fi와 같이 신뢰할 수 없는 네트워크에 사용된다. |

## 섹션 3 — 리눅스 CLI 기초 (Linux CLI Basics)

### 터미널이란

터미널(Terminal) = 텍스트 기반 인터페이스로, 기기를 제어하는 데 사용된다.  
리눅스 시스템에서는 그래픽 인터페이스 대신 명령어를 직접 입력하여 컴퓨터에 작업을 지시한다.

**터미널을 사용하는 이유**

- 여기저기 클릭하는 것보다 빠르다.
- 더 많은 제어권을 제공한다.
- 많은 보안 도구는 터미널에서만 실행된다.

---

### 파일 시스템 탐색 명령어

| 명령어 | 설명 |
|--------|------|
| `pwd` | 현재 위치한 폴더의 경로를 출력한다. (Print Working Directory의 약자) |
| `ls` | 현재 위치에 어떤 파일과 폴더가 있는지 확인한다. |
| `ls -l` | 현재 디렉터리의 내용을 자세한 정보(권한, 크기, 날짜 등)와 함께 나열한다. |
| `ls -al` | 숨김 파일을 포함하여 모든 파일을 자세히 나열한다. `.`으로 시작하는 파일은 리눅스가 기본적으로 숨기기 때문에 일반 `ls`로는 보이지 않는다. |
| `cd <directory>` | 지정한 디렉터리로 이동한다. |
| `cd ..` | 한 단계 상위(이전) 디렉터리로 돌아간다. |
| `find <starting_point> -name <filename>` | 파일 시스템 내에서 특정 파일을 찾을 때 사용한다. |
| `cat <filename>` | 파일의 내용을 읽어서 출력한다. |

**실습 1**: `find` 와 `cat`을 사용하여 `mission_brief.txt`의 경로를 찾고, 파일 안에 숨겨진 플래그 확인

---

### 시스템 정보 명령어

| 명령어 | 설명 |
|--------|------|
| `whoami` | 현재 로그인된 사용자의 이름을 출력한다. |
| `uname` | 현재 운영 체제의 이름만 출력한다. |
| `uname -a` | 커널 버전, 아키텍처 등 시스템에 대한 자세한 정보를 출력한다. |
| `df -h` | 디스크 사용량 및 사용 가능한 공간을 사람이 읽기 쉬운 단위(GB, MB)로 출력한다. |

**실습 2**: `whoami`로 현재 사용자 이름 확인 / `uname -a`로 커널 버전 확인 / `df -h`로 디스크 여유 공간 확인

---

### 중요 디렉터리

| 경로 | 설명 |
|------|------|
| `/etc` | 시스템 설정 파일과 구성 정보 파일이 저장되는 디렉터리 |

- **시스템 정보 검사**: `whoami`, `uname -a`, `df -h`
- **중요 구성 파일 읽기**: `/etc` 디렉터리 및 `cat` 명령어 활용

## 섹션 4 — 윈도우 CLI 기본 사항 (Windows CLI Basics)

### 파일 시스템 탐색 명령어

| 명령어 | 설명 |
|--------|------|
| `cd` | 인수 없이 입력하면 현재 위치한 디렉터리의 전체 경로를 출력한다. (리눅스의 `pwd`와 동일한 역할) |
| `cd <folder_name>` | 지정한 폴더로 이동한다. |
| `cd <path>` | 지정한 경로로 바로 이동한다. |
| `dir` | 현재 디렉터리에 있는 파일과 폴더 목록을 표시한다. |
| `dir /a` | 숨겨진 항목을 포함하여 모든 항목을 표시한다. |
| `dir /s task_brief.txt` | 현재 폴더 및 모든 하위 폴더를 검색하여 `task_brief.txt` 파일의 전체 경로를 표시한다. |
| `type <filename>` | 파일의 내용을 출력한다. (리눅스의 `cat`과 동일한 역할) |

**실습 1**: `dir /s task_brief.txt`로 파일 전체 경로 확인 → `cd`로 해당 폴더 이동 → `type task_brief.txt`로 내용 및 플래그 확인

---

### 시스템 정보 명령어

| 명령어 | 설명 |
|--------|------|
| `whoami` | 현재 로그인한 계정의 사용자 이름을 출력한다. |
| `hostname` | 컴퓨터 이름을 출력한다. |
| `systeminfo` | 운영 체제 버전, 설치 날짜, 메모리 등 시스템에 대한 자세한 정보를 출력한다. |
| `ipconfig` | 네트워크 인터페이스의 IP 주소, 서브넷 마스크, 게이트웨이 등 네트워크 구성 정보를 출력한다. |

**실습 2**: `whoami`로 사용자 이름 확인 / `hostname`으로 컴퓨터 이름 확인 / `systeminfo`로 OS 정보 확인 / `ipconfig`로 네트워크 정보 확인

## 섹션 5 — 운영 체제 보안 (Operating System Security)

### 운영 체제와 하드웨어의 관계

- **하드웨어**: 손으로 만질 수 있는 모든 컴퓨터 부품과 주변 장치
- **마더보드 (Motherboard)**: 컴퓨터의 핵심 부품. 키보드, 마우스, 모니터, 프린터 등 모든 하드웨어 부품이 연결된다.
- 모든 프로그램과 애플리케이션은 하드웨어에서 직접 실행될 수 없고, **운영 체제 위에서 실행**된다.
- 운영 체제는 특정 규칙에 따라 프로그램들이 하드웨어에 접근할 수 있도록 허용한다.

**운영 체제 종류**

| 분류 | 예시 |
|------|------|
| **개인용 데스크톱·노트북** | MS Windows 11, macOS |
| **스마트폰** | Android, iOS |
| **서버** | MS Windows Server 2022, IBM AIX, Oracle Solaris |
| **개인용 + 서버 겸용** | Linux (Ubuntu, Debian, Fedora 등) |

---

### 운영 체제에 저장되는 민감 데이터

스마트폰이나 컴퓨터에는 다양한 민감한 데이터가 저장되어 있어 보안이 중요하다.

| 기기 | 저장되는 민감 데이터 예시 |
|------|--------------------------|
| **스마트폰** | 가족·친구와의 사적인 대화, 개인 사진, 이메일, 저장된 비밀번호, 전자뱅킹 앱 |
| **노트북·컴퓨터** | 업무·학업 기밀 파일, 신분증·여권 사본, 이메일 클라이언트(Outlook 등), 저장된 비밀번호, 사진 |

---

### CIA 트라이어드 (보안의 세 가지 목표)

보안을 논할 때 반드시 보호해야 하는 세 가지 핵심 요소를 **CIA 트라이어드**라고 한다.

| 요소 | 영문 | 설명 |
|------|------|------|
| **기밀성** | Confidentiality | 비밀 및 개인 정보는 지정된 사람만 접근할 수 있어야 한다. |
| **무결성** | Integrity | 저장된 파일이나 전송 중인 데이터가 누구도 무단으로 변조할 수 없어야 한다. |
| **가용성** | Availability | 노트북이나 스마트폰은 필요할 때 언제든지 사용할 수 있어야 한다. |

---

### 공격자가 노리는 세 가지 취약점

1. **인증 및 취약한 비밀번호**
2. **취약한 파일 권한**
3. **악성 프로그램 (Malware)**

---

### 1. 인증 (Authentication)

**인증의 세 가지 방법**

| 유형 | 영문 | 예시 |
|------|------|------|
| **지식 기반** | Something you know | 비밀번호, PIN 코드 |
| **생체 인증** | Something you are | 지문, 얼굴 인식 |
| **소유 기반** | Something you have | OTP 문자 수신 전화번호, 보안 토큰 |

**취약한 비밀번호 — 상위 20위 (NCSC 발표)**

영국 국가사이버보안센터(NCSC, National Cyber Security Centre)는 가장 많이 사용되는 비밀번호 10만 개 목록을 발표했다.

| 순위 | 비밀번호 | 순위 | 비밀번호 |
|------|----------|------|----------|
| 1 | 123456 | 11 | 1234567890 |
| 2 | 123456789 | 12 | 123123 |
| 3 | qwerty | 13 | 000000 |
| 4 | password | 14 | iloveyou |
| 5 | 111111 | 15 | 1234 |
| 6 | 12345678 | 16 | 1q2w3e4r5t |
| 7 | abc123 | 17 | qwertyuiop |
| 8 | 1234567 | 18 | 123 |
| 9 | password1 | 19 | monkey |
| 10 | 12345 | 20 | dragon |

---

### 2. 파일 권한 (File Permissions)

적절한 보안은 **최소 권한 원칙 (Principle of Least Privilege)** 에 기반한다.  
사용자와 프로그램에는 작업 수행에 필요한 최소한의 권한만 부여해야 한다.  
파일 권한이 취약하면 공격자가 민감한 파일에 접근하거나 수정할 수 있다.

---

### 3. 악성 프로그램 (Malware)

| 유형 | 설명 |
|------|------|
| **트로이 목마 (Trojan Horse)** | 정상 소프트웨어처럼 위장하여 설치되면, 공격자에게 시스템 접근 권한을 부여한다. 공격자는 이를 통해 파일을 읽거나 수정할 수 있다. |
| **랜섬웨어 (Ransomware)** | 사용자의 파일을 암호화하여 열 수 없게 만들고, 복호화 키를 빌미로 몸값(ransom)을 요구한다. 파일이 암호화되면 키 없이는 내용을 알아볼 수 없는 형태가 된다. |

---

### 실습 — SSH 원격 접속 및 비밀번호 크래킹

```bash
# 1. SSH로 원격 시스템에 접속
ssh USERNAME@IP_ADDRESS
# 비밀번호 입력 후 접속

# 2. johnny 계정으로 전환
su - johnny
# johnny의 비밀번호 입력 후 접속

# 3. 명령어 실행 기록 확인 (root 비밀번호 힌트 포함)
history

# 4. root 계정으로 전환 후 플래그 확인
su - root
cat flag.txt
```

**실습 목표**
- 상위 7개 비밀번호를 바탕으로 johnny의 비밀번호 추측
- `history` 명령어로 root 비밀번호 확인
- root로 전환 후 `flag.txt` 내용 확인

> `history` 명령어는 이전에 실행한 모든 명령어 기록을 출력한다. 사용자가 평문으로 비밀번호를 명령어에 입력한 기록이 남아 있을 경우 그대로 노출된다.
