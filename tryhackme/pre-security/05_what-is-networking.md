## 섹션 1 — 네트워킹 기초 (Networking Basics)

### Task 1 — 네트워킹이란 무엇인가?

네트워크란 **연결된 것들**을 의미한다.

**현실 속 네트워크 예시**

| 예시 | 설명 |
|------|------|
| 대중교통 시스템 | 도시의 버스·지하철 노선망 |
| 국가 전력망 | 전력 공급을 위한 인프라 |
| 이웃 관계 | 이웃과 만나 인사하는 사람들의 연결 |
| 우편 시스템 | 편지·소포를 보내는 네트워크 |

**컴퓨팅에서의 네트워크**

| 항목 | 내용 |
|------|------|
| 정의 | 기술 기기들이 분산·연결된 구조 |
| 규모 | 최소 2개 장치 ~ 수십억 개 장치 |
| 포함 장치 | 노트북, 휴대폰, 보안 카메라, 신호등, 농업 장비 등 |
| 중요성 | 현대 사회와 사이버 보안의 핵심 개념 |

**예시**: Alice, Bob, Jim이 서로 메시지를 주고받으며 네트워크를 형성

| 퀴즈 | 정답 |
|------|------|
| 서로 연결된 기기들을 지칭하는 핵심 용어는? | `Network` |

### Task 2 — 인터넷이란 무엇인가?

**인터넷 정의**: 작은 네트워크들이 모여 이뤄진 **거대한 네트워크**

**인터넷의 역사**

| 연도 | 사건 |
|------|------|
| 1960년대 | ARPANET 프로젝트 시작 (미국 학술·군사 목적) |
| 1989년 | Tim Berners-Lee가 **WWW (World Wide Web)** 개발 |
| 현재 | 전 세계 정보 저장·공유 플랫폼으로 사용 |

**네트워크 두 가지 유형**

| 유형 | 설명 |
|------|------|
| **사설 네트워크 (Private Network)** | 내부 기기들끼리만 연결 |
| **공용 네트워크 (Public Network)** | 인터넷을 통해 외부와 연결 |

> 인터넷 구조: 여러 개의 작은 사설 네트워크들이 인터넷(공용)을 통해 서로 연결됨

| 퀴즈 | 정답 |
|------|------|
| WWW을 개발한 사람은 누구인가? | `Tim Berners-Lee` |

### Task 3 — 네트워크상의 장치 식별

네트워크 장치는 두 가지 식별 수단을 가진다 (인간의 이름과 지문에 비유).

| 인간 | 장치 | 특징 |
|------|------|------|
| 이름 | IP 주소 | 변경 가능 |
| 지문 | MAC 주소 | 고유값 (단, 스푸핑 가능) |

---

#### IP 주소 (Internet Protocol Address)

| 항목 | 내용 |
|------|------|
| 역할 | 네트워크 상에서 호스트를 일정 시간 동안 식별 |
| 구조 | 4개의 **옥텟(Octet)** 으로 구성 (예: `192.168.1.1`) |
| 범위 | 각 옥텟은 0~255 |
| 제약 | 같은 네트워크 내 동일 IP 중복 사용 불가 |

**공용 IP vs 사설 IP**

| 구분 | 역할 | 예시 |
|------|------|------|
| **사설 IP (Private)** | 사설 네트워크 내 장치 식별 | 192.168.1.77 |
| **공용 IP (Public)** | 인터넷에서 장치 식별 (ISP 제공) | 86.157.52.21 |

> 같은 사설 네트워크의 두 기기는 사설 IP는 다르지만, 공용 IP는 동일하다

**장치 IP 예시**

| 장치 이름 | IP 주소 | 유형 |
|----------|---------|------|
| DESKTOP-KJE57FD | 192.168.1.77 | 사설 |
| DESKTOP-KJE57FD | 86.157.52.21 | 공용 |
| CMNatic-PC | 192.168.1.74 | 사설 |
| CMNatic-PC | 86.157.52.21 | 공용 |

**IPv4 vs IPv6**

| 버전 | 주소 수 | 특징 |
|------|--------|------|
| **IPv4** | 2³² ≈ 42억 9천만 개 | 현재 주로 사용, 고갈 문제 발생 |
| **IPv6** | 2¹²⁸ ≈ 340조 개 이상 | IPv4 부족 문제 해결, 효율성 향상 |

> 시스코(Cisco) 추산 (2021): 2021년 말 인터넷 연결 기기 약 500억 개 → IPv4 고갈 가속화

---

#### MAC 주소 (Media Access Control Address)

| 항목 | 내용 |
|------|------|
| 할당 주체 | 제조 공장 (마더보드 네트워크 인터페이스에 부여) |
| 형식 | 12자리 16진수, 두 자리씩 콜론(`:`)으로 구분 |
| 예시 | `a4:c3:f0:85:ac:2d` |

**MAC 주소 구조**

| 구분 | 자리 | 의미 |
|------|------|------|
| 앞 6자리 | `a4:c3:f0` | 네트워크 인터페이스 제조사 식별 |
| 뒤 6자리 | `85:ac:2d` | 고유 번호 |

**MAC 스푸핑 (MAC Spoofing)**

| 항목 | 내용 |
|------|------|
| 정의 | MAC 주소를 소프트웨어적으로 위조하는 행위 |
| 위험성 | 방화벽이 MAC 기반 신뢰 설정 시 우회 가능 |
| 실습 예시 | 카페 유료 Wi-Fi — Alice MAC은 허용, Bob MAC은 차단 → Bob이 Alice MAC으로 스푸핑하면 통과 |
| 보안 시사점 | MAC 주소만으로 신뢰를 판단하는 설계는 취약하다 |

| 퀴즈 | 정답 |
|------|------|
| "IP"는 무슨 약자인가? | `Internet Protocol` |
| IP 주소의 각 부분은 무엇이라고 불리나? | `Octet` |
| IPv4 주소는 몇 자리 숫자로 구성되어 있나? | `4` |
| "MAC"은 무슨 약자인가? | `Media Access Control` |
| MAC 스푸핑 실습 플래그 | `THM{YOU_GOT_ON_TRYHACKME}` |

### Task 4 — 핑 (Ping / ICMP)

| 항목 | 내용 |
|------|------|
| 정의 | 가장 기본적인 네트워크 도구 중 하나 |
| 프로토콜 | **ICMP** (Internet Control Message Protocol) |
| 목적 | 장치 간 연결 성능 및 연결 존재 여부 확인 |
| 동작 원리 | ICMP 에코 패킷 전송 → ICMP 에코 응답으로 시간 측정 |
| 지원 OS | Windows, Linux, macOS 기본 내장 |

**명령어**

```bash
ping <IP 주소 또는 웹사이트 URL>
```

**실행 예시**

```
cmnatic@CMNatic-THM-LAPTOP:~$ ping 192.168.1.254
PING 192.168.1.254 (192.168.1.254) 56(84) bytes of data.
64 bytes from 192.168.1.254: icmp_seq=1 ttl=63 time=2.18 ms
64 bytes from 192.168.1.254: icmp_seq=2 ttl=63 time=2.53 ms
64 bytes from 192.168.1.254: icmp_seq=3 ttl=63 time=2.15 ms
64 bytes from 192.168.1.254: icmp_seq=4 ttl=63 time=3.34 ms
64 bytes from 192.168.1.254: icmp_seq=5 ttl=63 time=10.3 ms
64 bytes from 192.168.1.254: icmp_seq=6 ttl=63 time=4.45 ms
^C
--- 192.168.1.254 ping statistics ---
6 packets transmitted, 6 received, 0% packet loss, time 5008ms
rtt min/avg/max/mdev = 2.152/4.160/10.313/2.864 ms
```

**결과 해석**

| 항목 | 값 | 의미 |
|------|-----|------|
| 전송 패킷 | 6개 | ICMP 에코 패킷 전송 수 |
| 수신 패킷 | 6개 | 응답 수신 수 |
| 패킷 손실 | 0% | 연결 정상 |
| 평균 응답 시간 | 4.16ms | 장치 간 지연 시간 |

| 퀴즈 | 정답 |
|------|------|
| ping은 어떤 프로토콜을 사용하나? | `ICMP` |
| 10.10.10.10으로 핑을 보내는 명령은? | `ping 10.10.10.10` |
| 8.8.8.8로 핑을 보내면 어떤 플래그가 표시되나? | `THM{I_PINGED_THE_SERVER}` |

## 섹션 1 요약

| 개념 | 핵심 내용 |
|------|----------|
| **네트워크** | 연결된 기기들의 집합 (최소 2개~수십억 개) |
| **인터넷** | 작은 네트워크들이 모여 이뤄진 거대한 공용 네트워크 |
| **WWW** | 1989년 Tim Berners-Lee 개발 |
| **사설/공용 네트워크** | 내부 전용 vs 인터넷 연결 |
| **IP 주소** | 네트워크 상 장치 식별, 4개 옥텟, 공용/사설 구분 |
| **IPv4** | 2³² ≈ 42억 주소 (고갈 중) |
| **IPv6** | 2¹²⁸ ≈ 340조 이상 (IPv4 대체) |
| **MAC 주소** | 12자리 16진수, 앞 6자리=제조사·뒤 6자리=고유번호, 스푸핑 가능 |
| **Ping / ICMP** | ICMP 프로토콜, 네트워크 연결 성능 확인 도구 |
